# Discussion Board

Welcome! This is an open space for feedback, suggestions, feature requests, and general discussion about this repository. Whether you're a fellow Jamf admin, an MDE wrangler, or just passing through — your input is appreciated.

## How to Participate

Leave a comment by [opening a new Discussion](../../discussions/new) or [creating an Issue](../../issues/new) on this repo. You can also reply to any existing thread.

## Topics

Here are some areas where feedback would be especially helpful, but don't feel limited to these — anything goes.

### Extension Attributes

- Are there additional MDE health fields you'd like to see as EAs?
- Have you run into edge cases where an EA returns unexpected values?
- Would EAs for other security tools (CrowdStrike, SentinelOne, etc.) be useful?

### Scripts

- Are there scripts here that you've adapted for your own environment? What did you change?
- Any scripts that didn't work as expected in your setup?
- Ideas for new scripts — what repetitive Jamf tasks are you automating (or wish you could)?

### Documentation

- Is anything in the README unclear or missing context?
- Would example Jamf Pro policy configurations be helpful?
- Are the parameter tables easy to follow?

### General

- How are you using this repo? Cherry-picking individual scripts, forking the whole thing, something else?
- What Jamf Pro version and macOS versions are you running these against?
- Any interest in a companion repo for Windows/Intune equivalents?

## Ground Rules

1. **Be respectful.** We're all here to learn and share.
2. **Be specific.** If you hit a bug, include your macOS version, Jamf Pro version, and any relevant logs.
3. **Share back.** If you've improved on something here, consider opening a PR.

## Contact

If you'd prefer to reach out directly, feel free to open an issue tagged with `question` or connect with me through my GitHub profile.

Thanks for stopping by — looking forward to hearing from you.
